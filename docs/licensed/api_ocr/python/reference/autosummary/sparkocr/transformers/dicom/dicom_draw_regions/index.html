classmethod get_transfer_syntax_based_on_dicom_tags(**kwargs)#

Tries to find if the original file syntax is supported for encoding of the pixels. All the listed transfer syntax is supported for decoding, but for encoding they can vary depending on pydicom version. To overcome this gap for support of all transfer syntax we try to

  1. Check if the original transfer syntax is supported.

  2. If any other transfer syntax can be used instead keeping the compression type same.

  3. Else we change the compression type (lossy -> lossless)

  4. Else we fail to find a match

Returns:

  • String if there are errors.

  • Tuple if there are no errors, the contents and length of all cases are same except the first index.

    1. (Compresssion, _, _, )

    2. (Non-Compression, _, _, )

    3. (Failed-Match, _, _, )

classmethod validate_input_dicom_tags(**kwargs)#

Validates basic attributes of the DICOM file such as Photometric Interpretation, Transfer Syntax, Samples Per Pixel, and other relevant metadata.

Parameters:

  • pydicom_version (-) Version of pydicom in use.

  • force (-) If True, forces use of a default compression type for unsupported Transfer Syntax.

  • kwargs (-) DICOM attributes to validate, including PhotometricInterpretation, TransferSyntaxUID, SamplesPerPixel, BitsAllocated, BitsStored, HighBit, and others.

Returns:

  • String if there are missing or invalid tags.

  • Boolean True if the compression is supported.

  • Boolean False if the compression is not supported.
